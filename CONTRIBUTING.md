# Contributing

Thanks for contributing to `cursor-skill-builder`.

## Local Setup

1. Clone repository
2. Open in Cursor
3. Use project structure:
   - `prompts/` for exercises and reusable prompts
   - `examples/` for daily outputs
   - `evals/` for scoring and checklists
   - `src/` for small tooling improvements

## Workflow

1. Create or update files for the current day
2. Keep changes focused and easy to review
3. Update `README.md` progress section when a day is completed
4. Commit with clear, outcome-focused messages

## Commit Message Style

Prefer concise messages like:
- `Complete Day 4 output evaluation with rubric scoring`
- `Complete Day 5 guardrails and planner refactor`
- `Update README with Day 5 completion and progress tracker`

## Pull Request Expectations

- Explain what changed and why
- Mention files touched
- Include manual verification notes (for example script output checks)
- Keep PR scope limited to one day theme where possible
